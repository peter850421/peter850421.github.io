Dandelion

Dandelion platform optimizes bandwidth usage by reallocating idle household bandwidth for other purposes, such as live streaming. It consists of an entrance server, boxes, and publishers.

Boxes are microprocessors distributed in households to collect unused bandwidth. Box location and system information are recorded in the entrance server's database. Boxes periodically send updates to the server, including bandwidth, memory space, and GEOIP info. Inactive boxes are removed from the database.

In the case of live streaming, casters have publishers installed on their local machines. Publishers request a list of suitable boxes from the entrance server and establish connections. Publishers randomly select a box to store files or segments of the live stream, recording this action in their database. When viewers access the caster's live streaming web page, the server queries the caster's publisher, returns the file locations to the viewer's browser, and redirects the browser to the box for retrieval.

By utilizing idle household bandwidth through boxes, the web server reduces network traffic, lowering the cost of data transfer.
